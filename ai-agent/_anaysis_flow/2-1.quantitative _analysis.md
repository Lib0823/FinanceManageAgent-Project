## 1. 개요

정량분석은 KIS API와 DART DB에서 수집한 원천 데이터를 pandas / NumPy로 가공하여 **7개 피처**를 산출하는 단계다. 산출된 피처는 감성분석 1개, 시계열 예측 3개와 합산되어 **Gemini API 입력 총 11개**로 사용된다.

| 구분 | 피처 수 | 출처 |
| --- | --- | --- |
| 정량분석 (KIS 기반) | 4개 | 매일 수집 |
| 정량분석 (DART 기반) | 3개 | 분기 1회 갱신 |
| 감성분석 | 1개 | KR-FinBERT |
| 시계열 예측 | 3개 | Prophet |
| **Gemini 입력 합계** | **11개** |  |

---

## 2. 수집 방식

### 2-1. KIS API

종목별 매일 호출하며 asyncio 병렬 처리로 수집 시간을 단축한다. 모의투자 환경 기준 초당 5건 호출 제한이 적용되므로 병렬 처리 시 이를 반드시 준수해야 한다.

### 2-2. DART DB

분기 1회 재무데이터를 수집하여 `stock_financial` 테이블에 저장한다. 매일 파이프라인 실행 시에는 추가 API 호출 없이 DB 조회만 수행한다. 재무지표는 중장기 펀더멘탈 판단 용도로 분기 데이터도 충분히 유효하다.

## 3. 원천 데이터

### 3-1. 전날 분봉 (KIS API)

09:00~10:00 구간의 분봉 데이터를 수집한다. 시가·고가·저가·종가·거래량이 분 단위로 제공되며, 장초반 가격 흐름을 파악하는 데 사용된다.

### 3-2. 전날 일봉 (KIS API)

전날의 고가·저가·종가 데이터를 수집한다. 당일 가격 범위 내에서 종가가 어느 위치에 있었는지를 계산하는 데 사용된다.

### 3-3. 수급 데이터 (KIS API)

외국인과 기관의 순매수 금액을 수집한다. 1회 API 호출(`FHKST01010900`)로 외국인·기관 데이터를 동시에 가져올 수 있으며, 장 종료 후 확정치가 제공된다. 외국인·기관이 동시에 순매수하는 종목은 신뢰도 높은 수급 신호로 판단한다.

### 3-4. 재무제표 (DART DB)

금융감독원 전자공시시스템을 통해 분기별 재무데이터를 수집한다. PER·ROE·영업이익률 3개 지표를 활용하며, 단기 수급 흐름과 함께 기업 펀더멘탈을 동시에 반영하기 위해 포함된다.

## 4. 분석 방법 및 피처 산출 근거

### 4-1. 시간대별 집계 → `morning_return`

전날 분봉에서 09:00~10:00 구간을 집계하여 장초반 수익률을 산출한다.

**산출식**: (10:00 종가 - 시가) / 시가 × 100

장초반에 상승 출발했는지 하락 출발했는지는 당일 모멘텀을 반영하는 신호다. 기관투자자들의 프로그램 매매가 집중되는 시간대이므로 방향성이 명확하게 나타나는 경향이 있다. 양수이면 강한 매수세, 음수이면 매도 압력이 있는 것으로 해석한다.

### 4-2. 비율 계산 → `close_position`

전날 일봉의 고가·저가·종가로 종가의 고저 범위 내 위치를 산출한다.

**산출식**: (종가 - 저가) / (고가 - 저가)

0에 가까울수록 저가 마감(약세), 1에 가까울수록 고가 마감(강세)을 의미한다. 당일 매수세가 얼마나 강하게 마무리되었는지를 나타내며, 다음날 방향성 예측에 유효한 신호다.

### 4-3. 직접 사용 → `foreign_net_buy`, `institutional_net_buy`

KIS API에서 제공하는 원화 순매수 금액을 별도 계산 없이 그대로 피처로 사용한다.

외국인·기관 순매수는 시장에서 정보력이 높은 주체들의 자금 흐름을 나타낸다. 두 주체가 동시에 순매수하는 종목은 단독 순매수보다 신호 강도가 높다. 반대로 동시에 순매도하는 경우 하락 압력이 강한 것으로 해석한다.

### 4-4. 직접 사용 → `per`, `roe`, `operating_margin`

DART DB에서 조회한 분기값을 그대로 피처로 사용한다.

**PER**은 현재 주가가 이익 대비 고평가인지 저평가인지를 판단한다. 업종 평균 대비 낮은 PER은 저평가 신호로 해석될 수 있다. **ROE**는 자기자본 대비 이익 창출 능력을 나타내며, 높은 ROE는 효율적으로 자본을 운용하는 기업임을 의미한다. **영업이익률**은 본업에서 얼마나 수익을 내는지를 나타내며, 매출 규모와 무관하게 수익성의 본질을 보여준다.

## 5. 최종 피처 7개

### 5-1. KIS 기반 피처 (4개)

| 피처명 | 원천 데이터 | 의미 | 해석 방향 |
| --- | --- | --- | --- |
| `morning_return` | 전날 분봉 | 장초반 모멘텀 방향 | 양수 → 매수세, 음수 → 매도 압력 |
| `close_position` | 전날 일봉 | 당일 매수세 마무리 강도 | 1에 가까울수록 강세 |
| `foreign_net_buy` | 수급 데이터 | 외국인 자금 방향 | 양수 → 순매수, 음수 → 순매도 |
| `institutional_net_buy` | 수급 데이터 | 기관 자금 방향 | 양수 → 순매수, 음수 → 순매도 |

### 5-2. DART 기반 피처 (3개)

| 피처명 | 원천 데이터 | 의미 | 해석 방향 |
| --- | --- | --- | --- |
| `per` | 재무제표 | 가격 고평가 여부 | 낮을수록 저평가 |
| `roe` | 재무제표 | 기업 수익 창출력 | 높을수록 우량 |
| `operating_margin` | 재무제표 | 본업 수익성 | 높을수록 안정적 |

## 6. DB 저장 구조

피처값은 당일 계산 후 변수로 유지되어 Gemini 프롬프트에 직접 전달된다. 재무지표는 분기 단위로 `stock_financial` 테이블에 저장하며 매일 조회한다.

### stock_financial

DART API로 수집한 분기 재무지표를 저장한다. 매일 파이프라인에서는 이 테이블을 조회만 한다.

| 컬럼명 | 타입 | 설명 |
| --- | --- | --- |
| stock_code | VARCHAR(10) | 종목 코드 |
| base_date | DATE | 공시 기준일 |
| per | DECIMAL | 주가수익비율 |
| roe | DECIMAL | 자기자본이익률 (%) |
| operating_margin | DECIMAL | 영업이익률 (%) |

## 7. Gemini 입력 컨텍스트 구성

정량 7개 + 감성 1개 + 시계열 3개 = 총 11개 피처가 Gemini에 전달된다.

| 피처 그룹 | 피처 목록 |
| --- | --- |
| 정량 (KIS 기반) | morning_return, close_position, foreign_net_buy, institutional_net_buy |
| 정량 (DART 기반) | per, roe, operating_margin |
| 감성 | sentiment_score |
| 시계열 | prophet_price_trend, prophet_volume_trend, prophet_price_uncertainty |

각 피처 그룹이 Gemini 판단에서 담당하는 역할은 다음과 같다.

| 질문 | 담당 피처 |
| --- | --- |
| 돈이 들어오고 있나? | foreign_net_buy, institutional_net_buy, morning_return |
| 가격이 어떻게 움직였나? | close_position, prophet_price_trend |
| 앞으로 어떻게 될 것 같나? | prophet_price_trend, prophet_volume_trend, prophet_price_uncertainty |
| 기업 자체가 괜찮나? | per, roe, operating_margin |
| 뉴스 분위기는 어떤가? | sentiment_score |
