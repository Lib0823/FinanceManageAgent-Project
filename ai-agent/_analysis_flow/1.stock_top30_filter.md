---

## 1. 개요

종목 필터링은 코스피 100개 종목 풀에서 매일 분석 대상 30개 종목을 선정하는 단계다. KIS API로 수집한 4개 지표를 Scikit-learn StandardScaler로 정규화하고 가중합산 스코어를 산출하여 상위 30개를 확정한다.

| 항목 | 내용 |
| --- | --- |
| 입력 풀 | 코스피 시가총액 상위 100개 (하드코딩 고정) |
| 수집 지표 | 외국인 순매수, 기관 순매수, 거래량 배율, 가격 변동성 |
| 정규화 | Scikit-learn StandardScaler (매일 새로 fit) |
| 출력 | 분석 대상 종목 30개 |

---

## 2. 코스피 100 풀 관리

분석 대상 후보군을 코스피 시가총액 상위 100개 종목으로 고정한다. 종목코드를 하드코딩으로 관리하며, 코스피 100 구성 종목은 분기 단위로 변동이 발생하지만 기말 프로젝트 범위에서는 고정값으로 운영한다.

100개 종목을 고정하는 이유는 두 가지다. 첫째, 매일 동일한 종목 집합을 대상으로 수집하므로 데이터 일관성이 보장된다. 둘째, Scikit-learn StandardScaler가 매일 동일한 100개 기준으로 정규화하므로 일자 간 스코어 비교가 의미있어진다.

---

## 3. 수집 지표

파이프라인 실행 시 코스피 100개 종목 각각에 대해 KIS API를 호출하여 4개 지표를 수집한다.

```
100개 종목 × KIS API 호출
외국인 순매수 + 기관 순매수: FHKST01010900 (1회 호출로 동시 수집)
일봉 거래량 + 고가·저가:    일봉 조회 (1회 호출로 동시 수집)
→ 종목당 2회 호출 × 100종목 = 200회
→ 초당 5건 제한 → 약 40초 소요
```

### 3-1. 외국인 순매수 금액

외국인이 해당 종목을 얼마나 순매수했는지를 원화 금액으로 수집한다. 양수면 순매수, 음수면 순매도를 의미한다. 절대값이 클수록 외국인 자금의 움직임이 강하다는 신호다.

### 3-2. 기관 순매수 금액

기관이 해당 종목을 얼마나 순매수했는지를 원화 금액으로 수집한다. 외국인과 기관이 동시에 같은 방향으로 움직이는 종목은 수급 신호 강도가 높다.

### 3-3. 거래량 배율

전날 거래량을 20일 평균 거래량으로 나눈 값이다. 평소보다 거래량이 얼마나 터졌는지를 나타내며, 1.0이 평균 수준이고 값이 클수록 이상 수급이 발생한 것으로 판단한다.

**산출식**: 전날 거래량 / 20일 평균 거래량

20일 평균 거래량은 일봉 데이터 20일치에서 rolling mean으로 계산한다.

### 3-4. 가격 변동성

전날 고가와 저가의 범위를 저가 대비 비율로 표현한다. 값이 클수록 당일 가격 움직임이 컸다는 의미로, 시장의 관심이 집중된 종목을 포착하는 데 활용된다.

**산출식**: (고가 - 저가) / 저가

## 4. Scikit-learn StandardScaler 정규화

4개 지표는 단위와 스케일이 서로 달라 그대로 합산할 수 없다. 외국인 순매수는 원화 금액(수백억 단위), 거래량 배율은 1~10 사이 실수, 가격 변동성은 0~0.1 수준으로 스케일 차이가 크다.

StandardScaler는 각 지표를 평균 0, 표준편차 1로 변환하여 동일한 스케일에서 비교할 수 있게 만든다.

**정규화 방식**: 외국인·기관 순매수는 절대값으로 변환 후 정규화한다. 순매수(양수)와 순매도(음수) 모두 "강한 움직임"으로 해석하기 위해서다. 거래량 배율과 가격 변동성은 값 그대로 정규화한다.

**fit 방식**: 매일 파이프라인 실행 시 당일 100개 종목 데이터를 기준으로 새로 fit한다. 이는 "오늘 100개 종목 중 상대적으로 얼마나 강한가"를 기준으로 스코어를 산출하기 위해서다. 어제 기준이 아닌 오늘 시장 상황 기준으로 정규화하는 것이 종목 간 상대 비교에 더 적합하다.

---

## 5. 가중합산 스코어 산출

정규화된 4개 지표를 가중합산하여 종목별 최종 스코어를 산출한다.

**산출식**:

```
score = |외국인 순매수| × 0.3
      + |기관 순매수|   × 0.3
      + 거래량 배율     × 0.3
      + 가격 변동성     × 0.1
```

**가중치 근거**

| 지표 | 가중치 | 근거 |
| --- | --- | --- |
| 외국인 순매수 | 0.3 | 정보력 높은 주체의 자금 방향 |
| 기관 순매수 | 0.3 | 정보력 높은 주체의 자금 방향 |
| 거래량 배율 | 0.3 | 수급 없이 거래량만 터진 종목도 포착 |
| 가격 변동성 | 0.1 | 보조 신호 · 단독으로는 의미 약함 |

수급(외국인·기관)과 거래량에 각 30%씩 동일 가중치를 부여한 것은 수급이 없어도 거래량이 터진 종목은 반드시 분석 대상에 포함되어야 한다는 판단이다.

---

## 6. 상위 30개 선정

100개 종목을 스코어 내림차순으로 정렬하여 상위 30개를 최종 분석 대상으로 확정한다.

스코어가 높은 종목은 외국인·기관 수급과 거래량이 동시에 강한 종목으로, 매수세와 매도세 모두에서 움직임이 큰 종목이 선정된다. 이후 방향 태깅 단계에서 매수 후보와 매도 후보로 분류된다.

## 7. DB 저장 구조

### stock_filter_score

매일 100개 종목의 지표값과 선정 여부를 저장한다.

| 컬럼명 | 타입 | 설명 |
| --- | --- | --- |
| stock_code | VARCHAR(10) | 종목 코드 |
| trade_date | DATE | 거래일 |
| foreign_net_buy | BIGINT | 외국인 순매수 금액 (원) |
| institution_net_buy | BIGINT | 기관 순매수 금액 (원) |
| volume_ratio | DECIMAL | 거래량 배율 |
| price_volatility | DECIMAL | 가격 변동성 |
| final_score | DECIMAL | 가중합산 스코어 |
| is_selected | BOOLEAN | 상위 30개 선정 여부 |
